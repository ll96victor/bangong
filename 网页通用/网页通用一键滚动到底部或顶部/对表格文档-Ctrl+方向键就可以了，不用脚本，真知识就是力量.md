最后告诉我：
    ---

    **飞书多维表格（Bitable）在 `/wiki/` 路径下，表格内容被渲染在一个独立的 Shadow DOM 或动态注入的子框架中，外层 JS 完全无法穿透访问其内部结构。** 具体证据如下：

    - 主页面：canvas=1，但 bitable 容器内 canvas=0，说明 canvas 不在 bitable 容器的 DOM 子树里
    - iframe：body 完全为空（0x0）
    - 所有容器 scrollH === offsetH，没有任何可操作的滚动溢出
    - 没有任何虚拟滚动的 DOM 特征（absolute 定位大容器、transform 行偏移等）

    这意味着**油猴脚本从外部注入 JS 的方式，在飞书知识库内嵌多维表格这个场景下，存在无法突破的技术壁垒**。

    ---

    ## 实际可行的替代方案

    既然脚本路线走不通，有两个真正能用的方法：

    **方案一：直接用飞书自带的快捷键**

    飞书多维表格原生支持键盘跳转，在表格内点击任意单元格后：

    | 操作 | 快捷键 |
    |------|--------|
    | 跳转到第一行 | `Ctrl + Home` |
    | 跳转到最后一行 | `Ctrl + End` |
    | 跳转到首列 | `Ctrl + ←` |
    | 跳转到末列 | `Ctrl + →` |

    **方案二：改用飞书独立表格页面（非 wiki 内嵌）**

    直接打开 `xxx.feishu.cn/sheets/` 或 `xxx.feishu.cn/base/` 路径的表格，而不是通过知识库 `/wiki/` 访问。独立页面的渲染架构不同，脚本有更高概率能生效。如果你愿意，可以把独立表格的 URL 发给我，我针对那个路径重新诊断和适配。
