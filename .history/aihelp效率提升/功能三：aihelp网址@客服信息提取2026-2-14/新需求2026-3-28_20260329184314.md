## 1. 先确认：endpoint 本身没问题
官方文档里，MiMo 的 OpenAI 兼容地址就是：
```text
https://api.xiaomimimo.com/v1/chat/completions
```
在服务端（Python / Node / curl）都可以直接调，这点很多教程都验证过了。
所以你填的 endpoint 地址是对的，**问题不是“地址写错”，而是“浏览器环境不允许这样直接跨域请求”**。
---
## 2. 为什么在油猴里直连 `https://api.xiaomimimo.com` 不生效？
关键点：**浏览器同源策略（SOP）**。
- 你当前网页是：`https://xxx.com/...`
- MiMo API 是：`https://api.xiaomimimo.com/...`
  → 协议、域名都不同 → 浏览器认为是跨域请求。
- 对这种跨域 AJAX 请求，浏览器要求目标服务器在响应头里返回 CORS 头，例如：
  ```http
  Access-Control-Allow-Origin: https://xxx.com
  # 或
  Access-Control-Allow-Origin: *
  ```
- 如果服务器没返回这些头，**浏览器会直接拦截响应，JS 拿不到任何结果**，网络面板里能看到请求，但代码报错。
小米 MiMo API 是一个典型的 **后端 API 服务**，目前没有公开文档说明它支持浏览器端 CORS，示例也都是 curl / Python / 服务端调用。  
所以**默认应该认为：它不允许前端网页直接跨域访问**。
在油猴脚本里，如果你写：
```js
// 普通页面 JS
fetch('https://api.xiaomimimo.com/v1/chat/completions', { ... })
  .then(r => r.json())
  .then(console.log);
```
或者：
```js
const xhr = new XMLHttpRequest();
xhr.open('POST', 'https://api.xiaomimimo.com/v1/chat/completions');
xhr.setRequestHeader('Content-Type', 'application/json');
xhr.send(JSON.stringify({ ... }));
```
都会因为 CORS 被浏览器拦截，**功能看起来就是“不起作用”**。
---
## 3. 正确姿势：在油猴里用 `GM_xmlhttpRequest` 跨域
油猴提供了专门的跨域接口：`GM_xmlhttpRequest`，它可以绕过同源策略，**前提是你在脚本头部声明了 `@connect` 和 `@grant`**。
### 3.1 油猴脚本头部必须这样写
```js
// ==UserScript==
// @name         MiMo 聊天助手（示例）
// @namespace    http://tampermonkey.net/
// @version      0.1
// @description  在当前页面调用小米 MiMo API
// @author       you
// @match        https://xxx.com/*   // 你要注入的那个网站
// @grant        GM_xmlhttpRequest
// @connect      api.xiaomimimo.com
// ==/UserScript==
```
关键两行：
```js
// @grant        GM_xmlhttpRequest
// @connect      api.xiaomimimo.com
```
- `@grant GM_xmlhttpRequest`：告诉油猴你要用跨域请求接口。
- `@connect api.xiaomimimo.com`：显式允许脚本访问 `api.xiaomimimo.com` 这个域。
### 3.2 一个最小可用示例（直接复制改）
下面是一个**完整油猴脚本示例**，在当前页面调 MiMo Chat Completions，并在控制台输出结果：
```js
// ==UserScript==
// @name         MiMo Chat Example
// @namespace    http://tampermonkey.net/
// @version      0.1
// @description  调用小米 MiMo API 示例
// @author       you
// @match        https://xxx.com/*   // 改成你要注入的网站
// @grant        GM_xmlhttpRequest
// @connect      api.xiaomimimo.com
// ==/UserScript==
(function () {
  'use strict';
  // 你的 MiMo API Key
  const MIMO_API_KEY = '你自己的 API Key';
  // 要发送的消息
  const body = {
    model: 'mimo-v2-flash',   // 或 mimo-v2-pro
    messages: [
      { role: 'user', content: '你好，介绍一下小米 MiMo' }
    ],
    max_tokens: 1024,
    temperature: 0.7
  };
  GM_xmlhttpRequest({
    method: 'POST',
    url: 'https://api.xiaomimimo.com/v1/chat/completions',
    headers: {
      'Content-Type': 'application/json',
      'api-key': MIMO_API_KEY          // 或 Authorization: Bearer xxx
    },
    data: JSON.stringify(body),
    onload: function (response) {
      console.log('MiMo 响应状态：', response.status);
      console.log('MiMo 响应内容：', response.responseText);
      try {
        const result = JSON.parse(response.responseText);
        const content = result.choices?.[0]?.message?.content;
        if (content) {
          console.log('MiMo 回复：', content);
          // 这里你可以把 content 插到页面 DOM 里
        }
      } catch (e) {
        console.error('解析 JSON 失败：', e);
      }
    },
    onerror: function (error) {
      console.error('MiMo 请求失败：', error);
    }
  });
})();
```
要点说明：
- **必须用 `GM_xmlhttpRequest`，不能用 `fetch` / `$.ajax` / `XMLHttpRequest`**。
- `url` 写完整 endpoint：`https://api.xiaomimimo.com/v1/chat/completions`。
- `headers` 里：
  - `Content-Type: application/json`
  - 认证头：`api-key: xxx` 或 `Authorization: Bearer xxx`（两种方式官方都支持）。
- `data` 必须是 JSON 字符串，所以用 `JSON.stringify(body)`。

## 4. 常见坑自查清单
你可以对照一下：
1. **有没有用 `GM_xmlhttpRequest`？**  
   如果脚本里是 `fetch` / `$.ajax` / 自己封装的 `xhr`，那就是 CORS 问题。
2. **有没有写 `@grant GM_xmlhttpRequest`？**  
   没写的话，油猴根本不会给你注入这个函数，`GM_xmlhttpRequest is not defined`。
3. **有没有写 `@connect api.xiaomimimo.com`？**  
   没写的话，油猴会弹框问你是否允许跨域，或者直接拒绝。
4. **请求方法 / URL 有没有写错？**  
   - method 必须是 `POST`
   - URL 要完整：`https://api.xiaomimimo.com/v1/chat/completions`
5. **请求头 `Content-Type` 和认证方式是否正确？**  
   - `Content-Type: application/json`
   - 认证头：`api-key: xxx` 或 `Authorization: Bearer xxx`。
6. **请求体格式是否正确？**  
   至少要有 `model` 和 `messages` 字段，例如：
   ```json
   {
     "model": "mimo-v2-flash",
     "messages": [{ "role": "user", "content": "你好" }]
   }
   ```

## 5. 如果还是不行，怎么进一步排查？
如果你把上面的示例脚本改好（API Key、`@match`）后装上，**在控制台还是没任何输出**，可以这样排查：
1. 打开 DevTools → Console，看有没有：
   - `GM_xmlhttpRequest is not defined` → 说明没拿到 `GM_xmlhttpRequest`，检查 `@grant`。
   - 其他 JS 错误 → 可能脚本自身有问题。
2. DevTools → Network → 筛 XHR：
   - 看有没有发往 `api.xiaomimimo.com` 的请求。
   - 如果有，看响应头里有没有 `Access-Control-Allow-Origin`。
   - 如果没有这个请求，说明脚本根本没执行或没发请求。
3. 在脚本最前面加一行调试：
   ```js
   console.log('MiMo 脚本已加载');
   ```
   刷新页面，看控制台有没有这一行，确认脚本是否注入成功。
