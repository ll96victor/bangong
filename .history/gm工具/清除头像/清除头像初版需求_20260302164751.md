# 清除头像冷却时间 - 油猴脚本需求文档

## 1. 背景与问题

在工单界面处理工单时，经常遇到需要清除玩家头像冷却时间的情况。当前操作流程繁琐，需要手动在多个页面间切换并输入信息，效率较低。

## 2. 当前操作流程

### 2.1 步骤概览

| 步骤 | 操作位置 | 操作内容 |
|------|----------|----------|
| 1 | 工单列表界面 | 点击工单，复制玩家账号信息（UID、ServerID） |
| 2 | GM工具页面 | 输入ServerID，选择匹配项 |
| 3 | GM工具页面 | 滚动找到"清除上传头像倒计时"选项 |
| 4 | GM工具页面 | 输入UID，点击执行按钮 |
| 5 | GM工具页面 | 等待结果确认 |

### 2.2 详细操作说明

**第一步：提取账号信息**
- 用户需要在工单列表界面点击某个工单后才能看到账号信息
- 在工单详情中复制玩家的账号信息
- 示例：`UID = 1186053970, ServerID = 13814`

**第二步：打开GM工具**
- 访问地址：`https://gm.moba.youngjoygame.com:8090/#/customer/banTool`
- **注意事项**：
  - 首次使用需要用户登录
  - 如果用户没有登录就进不去，需要做保护措施，避免用户忘记登录而无法打开GM工具网址
- 点击左上角的服务器ServerID文本框
- 输入ServerID（如 `13814`）
- 在弹出的匹配项中点击对应选项（有时需手动筛选唯一匹配项）

**第三步：定位功能入口**
- 滚动鼠标滚轮下滑
- 找到"清除上传头像倒计时"选项

**第四步：执行清除操作**
- 点击"清除上传头像倒计时"下方的文本框
- 输入UID对应的值（如 `1186053970`）
- 点击"执行"按钮

**第五步：确认结果**
- 等待网页提示"loading"
- 出现文本 `Result:Ok` 则代表成功

## 3. 自动化需求

### 3.1 功能需求

生成一个油猴脚本，实现用户操作的自动化：

1. **触发方式**：用户点击脚本状态栏时执行
2. **信息提取**：从工单列表界面提取玩家账号信息（UID和ServerID）
3. **自动操作**：打开GM工具网页，自动完成所有操作步骤

### 3.2 技术需求

| 需求项 | 要求 |
|--------|------|
| 脚本类型 | 油猴脚本（Tampermonkey） |
| 触发机制 | 点击状态栏触发，非点击时不执行（避免内存占用） |
| 页面适配 | 仅在工单列表界面运行 |
| 信息提取 | 点击后立即提取，无需用户额外输入 |
| 代码注释 | 所有代码必须添加详细注释，确保用户能完全看懂 |

### 3.3 约束条件

| 约束 | 说明 |
|------|------|
| 运行页面 | 仅工单列表界面 |
| 工单界面地址 | `https://ml-panel.aihelp.net/dashboard/#/manual/tasks?searchType=7&SmartViewID=25285&SmartViewType=4` |
| 触发时机 | 用户点击状态栏时 |
| 执行方式 | 点击后立即提取账号信息 |

### 3.4 兼容性要求

本脚本需要与现有脚本 `工单助手与Task客服信息提取合并版 6.4.1` 共存，需注意：

1. **状态栏隔离**：使用不同的状态栏位置或合并到现有状态栏中
2. **存储键名隔离**：localStorage 键名添加唯一前缀，避免冲突
3. **DOM元素ID隔离**：所有创建的DOM元素使用唯一前缀
4. **事件监听隔离**：避免与现有脚本的事件监听器冲突

### 4.2 账号信息提取

**提取位置**：账号信息位于工单的"内部描述"区域中

**提取方式**（两种方式可选）：

**方式一：Ctrl+F 搜索方式**
```javascript
// 在页面文本中搜索 UID 和 ServerID
const uidPattern = /UID\s*=\s*(\d+)/i;
const serverIdPattern = /ServerID\s*=\s*(\d+)/i;
```

**方式二：参考现有脚本的提取逻辑**
参考 `工单助手与Task客服信息提取合并版 6.4.1` 中的 `extractInternalDescription` 函数：
1. 查找"内部描述"标签元素
2. 找到其相邻的内容区域
3. 提取文本内容
4. 使用正则匹配 UID 和 ServerID

**提取正则**：
```javascript
// UID 匹配：UID = 1186053970
const uidMatch = text.match(/UID\s*=\s*(\d+)/i);

// ServerID 匹配：ServerID = 13814
const serverIdMatch = text.match(/ServerID\s*=\s*(\d{4,5})/i);
```

### 4.3 GM工具打开方式（三种方案对比）

| 方案 | 优点 | 缺点 | 推荐度 |
|------|------|------|--------|
| **新标签页** | 实现简单，不影响当前页面 | 用户可能不注意到新标签页 | ⭐⭐⭐ |
| **iframe嵌入** | 在当前页面内操作，体验流畅 | 跨域限制，可能无法访问 | ⭐⭐ |
| **弹窗(window.open)** | 小窗口操作，不影响当前页面 | 可能被浏览器拦截 | ⭐⭐⭐ |

**建议**：优先使用新标签页方式，同时提供弹窗方式作为备选。
