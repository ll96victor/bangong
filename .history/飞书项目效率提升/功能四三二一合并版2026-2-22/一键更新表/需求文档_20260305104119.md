#问题需求

用户平时在win10电脑打开浏览器，然后打开一个工作项目链接，等工作项目网页加载完毕后使用提取信息油猴脚本提取必要的信息，并在在线文档粘贴所提取的信息。
鉴于用户需要处理多个工作项目链接，你有什么好的优化方法？

用户一次需要除了的链接数量不等，可能几条到60条不等。用户有一键打开多个链接的.py脚本，但是不敢一次打开60条。。。

用户暂时不想折腾API，想本地统一提取并整理成 CSV / Excel，再手动导入在线文档

对了，用户默认浏览器已经登录过一次了，之后打开的至多60条链接不用再登录。

## 方案B（Selenium）核心代码示例
这是最推荐的方案，因为：
- ✅ 直接复用你的油猴脚本逻辑
- ✅ 支持需要登录的页面
- ✅ 批量处理可控（分批、延迟）
```python
# -*- coding: utf-8 -*-
"""
批量提取工作项目信息工具
"""
import time
import pandas as pd
from selenium import webdriver
from selenium.webdriver.common.by import By
from selenium.webdriver.support.ui import WebDriverWait
from selenium.webdriver.support import expected_conditions as EC
from selenium.common.exceptions import TimeoutException
class ProjectInfoExtractor:
    def __init__(self, headless=False):
        """初始化浏览器"""
        options = webdriver.ChromeOptions()
        if headless:
            options.add_argument('--headless')  # 无头模式
        options.add_argument('--disable-gpu')
        options.add_argument('--no-sandbox')
        
        self.driver = webdriver.Chrome(options=options)
        self.wait = WebDriverWait(self.driver, 10)
        self.results = []
    
    def extract_single(self, url):
        """提取单个链接的信息 - 根据你的油猴脚本逻辑修改"""
        try:
            self.driver.get(url)
            
            # ========== 等待页面加载 ==========
            # 根据实际情况修改等待条件
            self.wait.until(EC.presence_of_element_located((By.CLASS_NAME, "your-content-class")))
            
            # ========== 提取信息 ==========
            # 将你的油猴脚本提取逻辑转换为Python代码
            info = {
                '链接': url,
                '项目名称': self.driver.find_element(By.CSS_SELECTOR, "h1.title").text,
                '项目编号': self.driver.find_element(By.CSS_SELECTOR, ".project-id").text,
                '负责人': self.driver.find_element(By.CSS_SELECTOR, ".owner").text,
                '状态': self.driver.find_element(By.CSS_SELECTOR, ".status").text,
                '创建时间': self.driver.find_element(By.CSS_SELECTOR, ".create-time").text,
                # ... 根据你的实际需求添加字段
            }
            
            return info
            
        except TimeoutException:
            print(f"⚠️ 超时: {url}")
            return {'链接': url, '错误': '加载超时'}
        except Exception as e:
            print(f"❌ 错误 {url}: {e}")
            return {'链接': url, '错误': str(e)}
    
    def extract_batch(self, urls, batch_size=10, delay=2):
        """批量提取，分批处理避免卡死"""
        total = len(urls)
        
        for i, url in enumerate(urls, 1):
            print(f"进度: [{i}/{total}] {url}")
            
            info = self.extract_single(url)
            self.results.append(info)
            
            # 延迟，避免请求过快
            time.sleep(delay)
            
            # 分批提示
            if i % batch_size == 0:
                print(f"━━━ 已完成 {i} 条，休息一下 ━━━")
                time.sleep(3)
        
        return self.results
    
    def save_to_file(self, filename='项目信息提取结果'):
        """保存结果到文件"""
        df = pd.DataFrame(self.results)
        
        # 同时保存 CSV 和 Excel
        df.to_csv(f'{filename}.csv', index=False, encoding='utf-8-sig')
        df.to_excel(f'{filename}.xlsx', index=False)
        
        print(f"✅ 已保存: {filename}.csv 和 {filename}.xlsx")
        return df
    
    def close(self):
        """关闭浏览器"""
        self.driver.quit()
# ==================== 使用示例 ====================
if __name__ == "__main__":
    # 1. 读取链接列表
    with open('链接列表.txt', 'r', encoding='utf-8') as f:
        urls = [line.strip() for line in f if line.strip()]
    
    print(f"共 {len(urls)} 个链接待处理\n")
    
    # 2. 创建提取器并开始提取
    extractor = ProjectInfoExtractor(headless=False)  # headless=True 可隐藏浏览器窗口
    
    try:
        # 先手动登录一次（如果需要）
        input("请在浏览器中完成登录，然后按回车继续...")
        
        # 批量提取
        extractor.extract_batch(urls, batch_size=10, delay=2)
        
        # 保存结果
        df = extractor.save_to_file(f'项目信息_{time.strftime("%Y%m%d_%H%M%S")}')
        print(f"\n提取完成，共 {len(df)} 条记录")
        
    finally:
        extractor.close()
```

### **核心方向：让油猴脚本从"单页提取"升级为"批量自动化流水线"**

用户已经有了 `.py` 脚本批量开链接、有油猴脚本提取信息，缺的只是把两者串联起来、解决"不敢一次开60条"的顾虑，以及把结果自动汇总成 CSV/Excel。下面按实际痛点逐一拆解。

---

### **第一个问题：为什么不敢一次开60条？**

根本原因通常是内存压力和浏览器卡死风险。解决思路不是"一次开60条"，而是**分批次、按节奏自动开**。

在现有的 `.py` 脚本里加入分批逻辑，比如每批开 10 条，每批之间等待一段时间（比如 15\~30 秒），让油猴脚本有足够时间在已打开的页面上完成提取，再开下一批。核心逻辑大致如下：

```python
import subprocess, time

links = [...]  # 你的全部链接列表
batch_size = 10
delay = 20  # 每批间隔秒数

for i in range(0, len(links), batch_size):
    batch = links[i:i+batch_size]
    for url in batch:
        subprocess.Popen(["start", "chrome", url], shell=True)
    print(f"已打开第 {i//batch_size + 1} 批，等待 {delay} 秒...")
    time.sleep(delay)
```

这样即使有 60 条链接，也是分 6 批、每批 10 条地平滑打开，浏览器压力始终可控。

### **第二个问题：油猴脚本提取后，结果怎么自动汇总？**

这是整个流程最关键的升级点。目前油猴脚本提取完信息后，用户需要手动复制粘贴，这在多页面场景下极其低效。改造方向是：**让油猴脚本把提取结果主动"存"下来，而不是等人来取**。

最实用的方案是利用浏览器的 `localStorage` 或 `GM_setValue` 作为临时数据池。每个页面的油猴脚本提取完信息后，把结果追加写入 `localStorage`（以 JSON 格式存储）。所有页面处理完毕后，再用一个独立的"导出页面"或脚本，把 `localStorage` 里积累的所有数据一次性读出来，生成 CSV 下载。

油猴脚本的提取+存储部分大致逻辑：

```javascript
// ==UserScript==
// @name         批量提取并暂存
// @match        https://your-project-site.com/*
// ==/UserScript==

(function() {
    // 提取你需要的字段
    const data = {
        title: document.querySelector('.project-title')?.innerText,
        id:    document.querySelector('.project-id')?.innerText,
        status: document.querySelector('.status')?.innerText,
        url:   location.href
    };

    // 读取已有数据，追加新条目
    const existing = JSON.parse(localStorage.getItem('extractedData') || '[]');
    existing.push(data);
    localStorage.setItem('extractedData', JSON.stringify(existing));
    console.log(`已暂存：${data.title}`);
})();
```

导出部分（可以是另一个油猴脚本，或者一个本地 HTML 文件）：

```javascript
const rows = JSON.parse(localStorage.getItem('extractedData') || '[]');
const csv = [Object.keys(rows[0]).join(',')]
    .concat(rows.map(r => Object.values(r).map(v => `"${v}"`).join(',')))
    .join('\n');

const blob = new Blob(['\uFEFF' + csv], {type: 'text/csv;charset=utf-8'});
const a = document.createElement('a');
a.href = URL.createObjectURL(blob);
a.download = 'projects_export.csv';
a.click();
```

注意开头的 `'\uFEFF'` 是 UTF-8 BOM，加上它 Excel 打开 CSV 时中文不会乱码。

### **第三个问题：如果 localStorage 跨域存不到同一个地方怎么办？**

`localStorage` 是按域名隔离的，如果工作项目链接都在同一个域名下，上面的方案完全没问题。如果涉及多个不同域名，则可以换用以下两种方式：

**方式一：GM_setValue（推荐）**，Tampermonkey 提供的跨页面存储，不受域名限制，所有脚本共享同一个数据空间：

```javascript
const existing = JSON.parse(GM_getValue('extractedData', '[]'));
existing.push(data);
GM_setValue('extractedData', JSON.stringify(existing));
```

导出时同样用 `GM_getValue('extractedData', '[]')` 读取即可。

**方式二：直接让油猴脚本调用本地 Python 服务**，油猴脚本用 `GM_xmlhttpRequest` 向本地跑的一个极简 Flask/http.server 发送提取结果，Python 端直接写入 CSV 或 Excel 文件。这个方案数据落盘更可靠，但需要每次工作前先启动本地服务，稍微麻烦一点。

整个流程串联起来是这样的：把所有链接放进 `.py` 脚本的列表 → 脚本分批（每批 10 条）自动打开浏览器 → 每个页面油猴脚本自动提取并追加存入 `GM_setValue` → 全部跑完后打开导出页面一键下载 CSV → 导入在线文档。

用户全程只需要：**启动 Python 脚本、等待自动处理、最后点一次导出**，不再需要对着每个页面手动操作。60条链接分6批跑完，浏览器压力完全可控。
