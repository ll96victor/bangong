# 一键更新表 - 需求文档

## 1. 背景与问题

### 当前工作流程
1. 打开浏览器 → 打开飞书项目链接 → 等待页面加载
2. 使用油猴脚本提取信息 → 手动粘贴到在线文档
3. 重复上述步骤 N 次（N 可能高达 60 次）

### 痛点
- 手动操作流程繁琐，重复劳动多
- 一次性打开 60 条链接会卡死浏览器
- 用户不想折腾 API，希望本地处理

### 目标
- 从「重复操作 60 次」变成「一键处理 60 条」
- 本地统一提取并整理成 CSV/Excel，再手动导入在线文档

## 3. 方案 A：Selenium 自动化（推荐）

### 3.1 方案概述
使用 Selenium 控制浏览器，自动完成：打开链接 → 等待加载 → 提取信息 → 保存结果

### 3.2 优势
- 直接复用油猴脚本的提取逻辑
- 支持需要登录的页面（浏览器已登录状态）
- 批量处理可控（分批、延迟、断点续传）

### 3.3 实现方案

#### 输入
- 链接列表文件（txt 格式，每行一个链接）
- 链接来源：通过油猴脚本的"列表链接提取"功能获得

#### 输出
- CSV 文件（UTF-8 BOM 编码，Excel 可直接打开）
- Excel 文件（可选）

#### 核心流程
```
1. 读取链接列表
2. 启动浏览器（复用已登录的 Chrome 配置）
3. 分批处理（每批 10 条，间隔 2-3 秒）
4. 对每个链接：
   - 打开页面
   - 等待关键元素加载完成
   - 提取信息（复用油猴脚本逻辑）
   - 追加到结果列表
5. 保存结果到 CSV/Excel
```

#### 关键技术点

**1. 复用浏览器登录态**
```python
options = webdriver.ChromeOptions()
options.add_argument('--user-data-dir=C:/Users/用户名/AppData/Local/Google/Chrome/User Data')
```

**2. 等待页面加载**
```python
# 等待页面标题包含项目名称
WebDriverWait(driver, 10).until(
    lambda d: '飞书项目' in d.title
)
```

**3. 提取逻辑转换（油猴 → Python）**

| 油猴代码 | Python 代码 |
|----------|-------------|
| `document.title` | `driver.title` |
| `document.querySelector('.class')` | `driver.find_element(By.CSS_SELECTOR, '.class')` |
| `el.textContent` | `el.text` |
| `el.getAttribute('href')` | `el.get_attribute('href')` |

**4. 断点续传**
- 记录已处理的链接
- 中断后可从上次位置继续

### 3.4 依赖
```
selenium>=4.0.0
pandas>=1.0.0
openpyxl>=3.0.0  # Excel 导出
```

## 4. 方案 B：油猴脚本升级（备选）

### 4.1 方案概述
改造现有油猴脚本，使用 `GM_setValue` 暂存数据，最后导出 CSV

### 4.2 实现方案

#### 步骤 1：改造提取功能
提取后不再复制到剪贴板，而是存储到 `GM_setValue`

```javascript
// 存储提取结果
const existing = JSON.parse(GM_getValue('extractedData', '[]'));
existing.push({
    name: name,
    status: '定位中未修复',
    date: formattedDate,
    ticketUrl: ticketUrl,
    processInfo: processInfo,
    url: location.href
});
GM_setValue('extractedData', JSON.stringify(existing));
```

#### 步骤 2：添加导出功能
新增一个功能区域，点击后导出所有暂存数据为 CSV

```javascript
function exportToCSV() {
    const rows = JSON.parse(GM_getValue('extractedData', '[]'));
    const csv = [Object.keys(rows[0]).join(',')]
        .concat(rows.map(r => Object.values(r).map(v => `"${v}"`).join(',')))
        .join('\n');
    
    const blob = new Blob(['\uFEFF' + csv], {type: 'text/csv;charset=utf-8'});
    const a = document.createElement('a');
    a.href = URL.createObjectURL(blob);
    a.download = 'projects_export.csv';
    a.click();
}
```

#### 步骤 3：配合批量打开脚本
1. 运行 `batch_open_links_improved.py` 分批打开链接
2. 每个页面自动提取并存储
3. 全部完成后点击导出

### 4.3 注意事项
- `GM_setValue` 需要在脚本头部声明 `@grant GM_setValue` 和 `@grant GM_getValue`
- 数据存储在浏览器本地，关闭浏览器后数据仍在
- 需要手动清除数据的功能
