## 2. 当前功能概览（v4.0.1）


| 按钮 | 功能 | 入口函数 |
|------|------|----------|
| 📁 | 一键更改分组为 `CN 二线-BUG` | `handleChangeGroup()` |

### 核心实现原则

- **最小改动**：保留原有复制、分组、标签主流程，只做增量扩展
- **整排拖拽**：拖动任意一个按钮，整排一起移动，位置保存在 `localStorage`
- **配置外置**：API Key / Endpoint / 模型不写死在脚本里，统一通过油猴菜单录入
- **浮动面板**：翻译、AI、日志都用轻量浮动面板展示，一次只开一个，避免叠层混乱
- **错误隔离**：翻译或 AI 失败不会影响复制、分组、标签三项已有功能

## 3. 本次升级内容（2026-03-28）

### 3.1 重命名

1. 脚本 `@name` 已改为：`客诉助手（原客服信息提取）`
2. 主维护文档名称统一为：`交接与维护文档`
3. `维护文档.md` 保留为兼容入口，提示已迁移到本文件

### 3.2 翻译模块补充优化（🌐）

- 默认用途：优先翻译待发送的客服回复，而不是优先翻译用户原话
- 默认内容：优先带出最新客服回复；若页面里没有，则允许用户直接粘贴自己的回复
- 客诉语种参考：优先根据最新用户消息识别客诉目标语种，并自动预选到下拉框
- 语种识别：`detectLanguage()` 调用 Google 免费翻译接口自动识别
- 翻译逻辑：`translateWithAutoDetection()` / `translateText()`
- 展示方式：浮动面板展示检测语种、目标语种、可编辑原文、译文
- 手动切换：面板里的语种下拉框可直接切换目标语种


### 3.3 AI 辅助模块补充优化（🤖）

- 上下文来源：
  - 用户消息：`.msg.msg-left`
  - 客服消息：`[data-testid="agentMessageItem"].msg.msg-right`
- 自动提取：最近 10 条上下文，避免内容过长
- 输出模式：
  - `推荐回复`
  - `优化当前回复`
- 自定义优化：面板新增“待优化回复”输入区，可直接粘贴自己写的回复再优化
- 说明语言：AI 输出固定附带中文说明，避免客服看不懂小语种结果
- 输出语言：面向用户的话术按目标语种输出，也可手动切换
- 主备方案：
  - 主：智谱 GLM
  - 备：小米 MiMo
- 回退机制：优先用面板中选中的方案，失败后自动切换到另一方案


### 3.4 新增日志模块（📜）

- 统一日志入口：`log()` / `logWarn()` / `logError()`
- 面板功能：查看、复制、清空日志
- 日志内容：记录按钮操作、翻译调用、AI 调用、失败原因
- 安全要求：不输出完整 API Key

### 3.5 拖拽改造

- 旧版：只能拖容器空白区域
- 新版：按住任意按钮，移动超过 `5px` 即判定为拖拽
- 点击与拖拽分离：
  - 小于阈值：执行按钮功能
  - 大于阈值：只移动整排按钮，不触发点击

## 4. API 选型与结论

### 4.1 翻译

| 用途 | 方案 | 说明 |
|------|------|------|
| 翻译 | Google 免费翻译 | 调用少、改动小、速度快，适合当前需求 |

### 4.2 AI 辅助

| 类型 | 方案 | 结论 |
|------|------|------|
| 主方案 | `GLM-4.7-Flash` | 适合纯文本话术生成，优先使用 |
| 备方案 | `GLM-4-Flash-250414` | 作为 GLM 兼容备选 |
| 备用服务 | `MiMo-V2-Pro` | 适合作为 OpenAI 兼容回退 |

### 4.3 明确不采用的模型

| 模型 | 原因 |
|------|------|
| `GLM-4.6V-Flash` | 多模态，当前不需要看图 |
| `GLM-4.1V-Thinking-Flash` | 过重，不适合当前轻量文本场景 |
| `GLM-4V-Flash` | 视觉模型，不相关 |
| `Cogview-3-Flash` | 图片生成，不相关 |
| `CogVideoX-Flash` | 视频生成，不相关 |
| `MiMo-V2-TTS` | 语音场景，不相关 |
| `MiMo-V2-Omni` | 多模态，当前非必要 |

| 项目 | 默认值 |
|------|--------|
| 默认目标语种 | `en` |
| GLM Endpoint | `https://open.bigmodel.cn/api/paas/v4/chat/completions` |

## 7. 使用方法

### 7.1 安装

1. 浏览器安装 Tampermonkey
2. 导入 `.user.js` 脚本
3. 进入客诉页面，确认右上区域出现 6 个按钮
4. 如需 AI 功能，先打开 Tampermonkey 菜单配置 GLM 或 MiMo

### 7.2 按钮操作

| 按钮 | 操作 | 说明 |
|------|------|------|
| 📁 | 单击 | 更改分组为 `CN 二线-BUG` |
| 📋 | 单击 | 复制 `URL@客服` |
| 🏷️ | 单击 | 添加 `BUG二綫 BUG Agents` 标签 |
| 🌐 | 单击 | 打开翻译面板，默认翻译客服回复，也可粘贴自定义内容 |
| 🤖 | 单击 | 打开 AI 面板并生成推荐/优化结果，结果附中文说明 |

| 📜 | 单击 | 打开或关闭日志面板 |
| 任意按钮 | 拖动 | 整排按钮一起移动，位置自动保存 |

## 8. 风险与兼容性

| 风险项 | 说明 | 应对 |
|--------|------|------|
| Google 翻译频率限制 | 免费接口不适合高频批量调用 | 当前日均 <100 次可接受 |
| DOM 结构变更 | 页面升级后消息选择器可能失效 | 对照 `AIhelp网页结构.md` 检查 `.msg-left` 与 `agentMessageItem` |
| AI Key 过期 | 小米免费期结束或 GLM Key 变更 | 在油猴菜单中重新配置 |
| MiMo Endpoint 不确定 | 用户环境可能不是统一域名 | 由用户手动填写完整 Endpoint，脚本不硬编码 |
| 面板重叠 | 多个面板同时打开会遮挡 | 当前策略是一次只保留一个面板 |
| 语种判断偏差 | 极短文本可能识别不准 | 面板允许手动切换目标语种 |

## 9. 常见问题排查

### 9.1 翻译无反应

1. 检查是否提取到了最新用户消息
2. 检查 `@grant GM_xmlhttpRequest` 是否存在
3. 检查网络是否能访问 `translate.googleapis.com`
4. 打开 `📜` 查看错误日志

### 9.2 AI 功能报错

1. 先确认至少配置了 GLM 或 MiMo 其中一个
2. 若用 MiMo，确认填写的是完整 Chat Completions Endpoint
3. 打开 `📜` 查看失败提供方与错误信息
4. 如果主方案失败，脚本会自动尝试另一方案

### 9.3 按钮拖不动或误触发

1. 当前拖拽阈值是 `5px`
2. 轻点会执行功能，拖动才会移动整排
3. 若位置异常，可清理 `localStorage` 中的 `ai-btn-container-position`
