# AIHelp工单批量筛选与处理工具 维护与交接文档

## 1. 用户需求

- 小图标点击后直接展开并显示日志内容，不再额外点“日志”按钮。
- 展开后隐藏小图标。
- 展开面板支持拖动。
- 删除“筛选AI识别为Bug的工单”单独入口，只保留 `BJ` 自动处理入口。
- 优化批量分配、批量解决这类操作速度，减少页面明明已加载却还要等很久的问题。
- 保留原“批量设为已解决”功能。
- 新增“批量设为已解决并发已知邮件”功能。
- 已知邮件用唯一匹配文本 `8 Noticed.mail`。

## 2. 原有功能

- 悬浮小图标点击后展开批量操作按钮面板。
- 独立日志面板，原来需要单独按钮打开。
- `BJ`：筛选 AI 识别为 Bug 的工单后自动解决。
- 原独立 `Bug` 筛选按钮。
- `MCGG` 标题筛选。
- `s57` 描述筛选。
- 清除筛选。
- 批量处理。
- `L / N / W / X` 批量分配。
- 批量设为已解决。

## 3. 本次修改

- 将“按钮面板”和“日志面板”合并成统一展开面板。
- 点击小图标后直接展开统一面板，并直接看到日志区域。
- 统一面板关闭前，小图标隐藏；关闭后恢复显示。
- 统一面板支持拖动和调整大小，继续复用原位置与大小记忆。
- 从动作数组中移除原 `ActionA` 单独入口。
- 从动作数组中移除原“日志”入口，日志改为统一面板直接显示。
- 新增 `KM` 按钮：批量设为已解决并发送 `8 Noticed.mail`。
- 为批量分配、批量解决、新增 `KM` 按钮补充公共下拉与提交按钮方法，改为“元素出现就继续”的更快链路。

## 4. 使用方法

- 点击页面右侧悬浮小图标，展开统一面板。
- 在统一面板中直接点对应按钮执行批量操作。
- 面板下半部分会持续显示日志。
- 关闭按钮会收起统一面板，并恢复悬浮小图标。
- `KM` 按钮用于“批量已解决并发已知邮件”。

## 5. 注意事项

- 这份脚本文件本身存在较多历史编码痕迹，后续修改时优先按“最小改动 + 局部覆盖”方式处理，不建议大范围重写旧块。
- 旧的 `ActionA`、`ActionLog`、旧展开面板函数目前主要作为历史保留代码，实际入口以最新动作数组和后置覆盖函数为准。
- 若后续要继续提速，优先复用本次新增的下拉/提交公共方法，不要再回到大量固定 `sleep`。
- 若页面的表单标题文本发生变化，优先检查：
  - `工单受理人`
  - `工单状态`
  - `发送奖励`
  - `提交 / 确认`

## 6. 本次确认信息

- “已知邮件”按用户确认，使用唯一匹配文本：`8 Noticed.mail`。
