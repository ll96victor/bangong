# 需求文档：新增批量分配功能

## 一、需求概述

为脚本 `AIHelp工单批量筛选与处理工具-3.1.0.user.js` 新增批量分配工单受理人的功能。

---

## 二、原脚本信息

- **脚本路径**：同级目录 `AIHelp工单批量筛选与处理工具-3.1.0.user.js`
- **当前版本**：3.1.0
- **UI结构**：悬浮图标（聚合图标）+ 点击展开的功能面板（3x2网格，6个按钮）+ 日志面板

---

## 三、功能需求

### 3.1 新增功能：批量分配受理人

#### 3.1.1 功能描述

用户手动勾选工单后，点击对应的受理人按钮，脚本自动完成以下操作：

1. 点击"编辑"按钮（用户勾选工单后已出现）
2. 等待批量编辑弹窗加载
3. 点击"工单受理人"下方的文本框
4. 在弹出的搜索框中输入指定的受理人名称
5. 点击弹窗中匹配的选项
6. 点击"提交"按钮完成分配

#### 3.1.2 受理人映射表

| 按钮显示 | 受理人ID | 日志显示 |
|---------|---------|---------|
| L | CN-Lianglei | 分配给 CN-Lianglei |
| N | CN-Niao | 分配给 CN-Niao |
| W | CN-Wumengru | 分配给 CN-Wumengru |
| X | CN-Xutingting | 分配给 CN-Xutingting |

#### 3.1.3 操作前置条件

- 用户已手动勾选需要分配的工单（一个或多个）
- "编辑"按钮已出现在页面上（勾选后自动出现）
- 用户**未**点击"编辑"按钮，弹窗未打开

## 八、技术实现要点

### 8.1 元素定位参考

根据原脚本 `ActionE`（批量处理）的实现，批量分配功能需要定位以下元素：

| 步骤 | 元素定位方式 |
|-----|------------|
| 点击编辑按钮 | `ToolUtil.clickByText(['button', 'span'], '编辑')` |
| 查找工单受理人输入框 | 通过 `.el-form-item__label` 包含"工单受理人"定位 |
| 搜索框 | `input[placeholder="搜索"]`，选择距离最近的 |
| 下拉选项 | `li` 元素，匹配受理人名称 |
| 提交按钮 | `button` 元素，文本为"提交" |

### 8.2 等待时间建议

| 操作 | 建议等待时间 |
|-----|------------|
| 点击编辑后等待弹窗 | 1500ms |
| 点击输入框后等待搜索框 | 800ms |
| 输入后等待下拉列表 | 800ms |
| 点击选项后 | 500ms |

### 8.3 错误处理

- 未勾选工单时，"编辑"按钮不存在，应提示用户先勾选工单
- 未找到受理人选项时，记录错误日志并提示
- 提交失败时，记录错误日志
