# 油猴脚本需求：AIHelp工单批量筛选与处理工具

## 背景与目标

用户经常需要在AIHelp工单管理页面执行重复性的筛选和批量操作，目标是开发一个油猴脚本，通过状态栏提供一键快捷操作，提升工作效率。

### 目标网页
```
https://ml-panel.aihelp.net/dashboard/#/manual/tasks?searchType=7&SmartViewID=25285&SmartViewType=4
```
**注意**：匹配的网页URL不能包含"ticket"字符串。

---

## 核心需求

### 1. 状态栏设计
- 在页面底部显示可移动的状态栏
- 状态栏包含6个功能区块，前5个对应5种筛选/操作功能，第6个是日志面板入口
- 悬浮时立即显示功能提示（tooltip），悬浮超过2秒显示详细消息
- 点击对应区块执行相应功能
- 详细规范参考：`油猴脚本状态栏规范2026-2-22 - 副本.md`和`rules2026-2-26优化 - 副本.md`


### 2. 网页交互规范
- 网页交互逻辑参考：`AIHelp_Automation_Expert_Fusion - 副本.md`
- 所有点击操作需要精确匹配元素，避免使用模糊选择器
- 需要处理页面加载等待和弹窗出现的时间延迟

## 技术要求

### 代码规范
1. 使用ES6+语法，确保兼容性和可维护性
2. 所有DOM操作使用`document.querySelector`或`document.querySelectorAll`
3. 元素选择器应尽可能精确，优先使用data属性、aria-label等语义化属性
4. 添加适当的等待逻辑处理异步加载（使用MutationObserver或setTimeout）
5. 实现错误捕获和日志记录，确保可调试性

### 用户体验
1. 状态栏使用固定定位，z-index高于页面其他元素
2. 悬浮提示使用延迟显示（立即显示简短提示，2秒后显示详细说明）
3. 操作执行时提供视觉反馈（如按钮 loading 状态）
4. 日志面板记录每次操作的执行结果和耗时

### 健壮性要求
1. 检测目标元素是否存在，如不存在则等待或提示用户
2. 处理页面跳转、弹窗关闭等异常情况
3. 支持重复执行，多次点击不产生副作用
4. 在控制台输出详细操作日志，便于排查问题

| 问题类型 | 具体问题 | 解决方案 |

| **框架兼容** | 直接设置value对Vue无效 | 使用原生setter |
| **文本匹配** | 匹配到包含多个按钮的父元素 | 精确匹配 `text === '提交'` |
