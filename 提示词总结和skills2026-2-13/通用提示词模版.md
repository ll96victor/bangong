### 性能优化要点

#### 场景A：优化现有脚本

**快速点击机制**：为点击功能添加快速点击的逻辑代码，如果快速点击不成功，再用原来的代码操作：

```javascript
async fastClick(element, options = {}) {
    const { fastDelay = 100, fallbackDelay = 300, logger = null } = options;
    
    // 快速尝试：元素已存在，立即点击
    if (element && this.isElementAvailable(element)) {
        console.log('[快速点击] 元素已存在，立即点击');
        if (logger) logger.log('快速点击成功');
        element.click();
        await this.sleep(fastDelay);
        return true;
    }
    
    // 快速尝试失败，使用原来的等待逻辑
    console.log('[快速点击] 元素不可用，等待后重试');
    if (logger) logger.log('等待后重试点击');
    await this.sleep(fallbackDelay);
    return false;
}
```

**等待时间双逻辑**：

| 模式 | 等待时间 | 说明 |
|-----|---------|------|
| 快速模式 | 100ms | 元素已存在时立即执行 |
| 回退模式 | 300-800ms | 保持原来的等待时间 |

#### 场景B：创建新脚本

**同时提供两种点击方式**：

```javascript
const ToolUtil = {
    // 快速点击（适合熟练用户）
    async fastClick(element, options = {}) {
        const { delay = 100 } = options;
        if (element && this.isElementAvailable(element)) {
            element.click();
            await this.sleep(delay);
            return true;
        }
        return false;
    },
    
    // 保守点击（适合新手用户，更稳定）
    async safeClick(element, options = {}) {
        const { delay = 500 } = options;
        await this.sleep(delay); // 先等待
        if (element && this.isElementAvailable(element)) {
            element.click();
            await this.sleep(delay);
            return true;
        }
        return false;
    }
};
```

**全面优化范围**：
- 点击文本框
- 点击文本框后弹出的搜索框
- 对应匹配项的点击速度

### 常见问题

#### Q1：如何获取元素的 outerHTML？

1. 按 F12 打开开发者工具
2. 点击左上角"选择元素"图标（或按 Ctrl+Shift+C）
3. 点击目标元素
4. 右键 → Copy → Copy outerHTML
5. 粘贴到提示中

#### Q2：如何判断弹窗类型？

查看元素的 class 名称：
- `el-dialog`：对话框
- `el-popover`：弹出框
- `el-drawer`：抽屉

#### Q3：如何判断编辑器类型？

查看元素结构：
- TinyMCE：包含 `<div class="tox tox-tinymce">` 和 `<iframe>`
- Quill：包含 `<div class="ql-editor">`
- 原生：`<textarea>` 或 `<input>`

#### Q4：为什么需要提供参考脚本？

参考脚本可以帮助AI：
1. 了解现有代码结构和风格
2. 复用已有的工具函数
3. 保持代码一致性
4. 避免重复造轮子
