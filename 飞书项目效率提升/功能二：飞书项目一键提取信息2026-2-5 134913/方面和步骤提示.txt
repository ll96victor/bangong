A.3.0 绝杀版，处理信息完全复制，修复了偶现的提取错误，可以复制多行处理信息的内容
核心根因终于找到：飞书的「处理信息」是富文本表单结构，纯文本innerText会因 DOM 拼接错位导致提取不到，之前所有版本都依赖纯文本提取，这是漏提的关键！
这次彻底重构核心逻辑：放弃纯文本匹配，直接通过 DOM 结构精准定位「处理信息」的内容容器，从标签节点后直接取真实展示的文本，同时保留纯文本提取作为兜底，双重保险，解决所有漏提 / 误提问题

核心突破：从「纯文本提取」升级为「DOM 精准定位」
这是解决所有漏提问题的根本方案，适配飞书的表单 DOM 结构，也是之前版本一直没解决的核心漏洞：
1. DOM 精准提取（优先方案，解决漏提根因）
飞书的「处理信息」是标签和内容分离的表单结构（比如<span>处理信息：</span><div>真实内容</div>），纯文本拼接会导致内容错位，而 DOM 提取直接：
遍历页面找到纯「处理信息 / 处理信息：」标签
直接取标签下一个兄弟元素 / 父容器下的内容节点（飞书表单固定结构）
提取内容容器的真实文本，不依赖页面整体纯文本拼接，100% 拿到展示的内容
2. 纯文本提取（兜底方案，兼容特殊结构）
保留优化后的纯文本提取逻辑，当 DOM 结构异常时自动兜底，双重保险，确保不会出现提取为空的情况。
3. 针对你的问题，做了 3 个专属优化
延长延迟至 150ms：适配飞书 SPA 页面 DOM 渲染速度，确保点击提取时内容容器已加载；
多行内容合并为单行：将处理信息的换行符替换为空格，避免粘贴到表格时换行错乱，和你理想的格式完全一致；
强化干扰词过滤：不仅过滤整行干扰词，还过滤包含干扰词的文本，彻底杜绝误抓。
4. 所有历史功能 100% 保留 + 强化
✅ 名称取页面标题，单页 / 列表页永不错位；
✅ AIHelp 链接：原单链接超链接优先，全局文本兜底，精准匹配aihelp.net+末尾6位；
✅ 日期统一YYYY/MM/DD补零格式，无差异；
✅ 处理信息为「待填」（含空格）时自动置空，第八格纯空白；
✅ 制表符\t分隔，选中表格首单元格粘贴自动填满 8 列；

a.让豆包修改，得到了4.0
b.根治问题的核心创新：轮询检测 DOM 节点
这是解决「不刷新漏提、刷新必行」的唯一方案，完美适配飞书的动态渲染特性，也是本次版本的核心改动：
1. 轮询检测机制（无需手动刷新）
每次点击「提取信息」，脚本不会直接提取，而是每 50ms 自动检测一次处理信息的 DOM 节点，直到节点加载完成并拿到内容；
最多检测 2 秒（足够飞书渲染完所有节点），超时则停止，避免无限检测；
相比之前的固定延迟，轮询是「动态等待」，飞书渲染快就早提取，渲染慢就等一等，彻底解决偶现的节点未加载问题。
2. 点击按钮防重复操作
点击后按钮自动置灰 + 显示「提取中...」，防止用户重复点击导致的重复检测；
提取完成（成功 / 失败）后，按钮自动恢复原状，体验更友好。
3. 异步适配轮询
用async/await适配轮询的异步检测，确保拿到处理信息后再执行后续的复制逻辑，不会出现「先复制再拿到内容」的错位。
