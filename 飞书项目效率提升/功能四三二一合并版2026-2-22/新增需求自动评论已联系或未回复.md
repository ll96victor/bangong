# 需求文档：自动评论功能（已联系/未回复）

## 1. 功能概述

### 1.1 用户场景
用户在飞书项目网页中频繁需要评论"已联系"或"联系玩家24小时后玩家未提供有效信息"，需要通过脚本自动化此操作。

### 1.2 功能目标
在现有脚本的状态栏中新增两个快捷评论图标，点击后自动完成评论发布流程。

---

## 4. 执行流程

### 4.1 主流程

```
用户点击状态栏图标
    ↓
判断当前页面状态（SPA处理）
    ↓
查找评论框
    ↓
├─ 未找到评论框 → 点击评论区域激活
├─ 找到但未激活 → 点击激活评论框
└─ 已激活 → 继续
    ↓
等待评论框完全加载（重试机制）
    ↓
输入评论内容
    ↓
点击发布按钮
    ↓
显示操作结果（成功/失败提示）
```

### 4.2 SPA页面处理

```javascript
// 检测是否在项目详情页
function isInProjectDetailPage() {
    return document.querySelector('.meego-comment') !== null;
}

// 监听页面变化（SPA）
function observePageChange(callback) {
    const observer = new MutationObserver((mutations) => {
        // 检测评论区域是否出现
        if (document.querySelector('.meego-comment')) {
            callback();
        }
    });
    observer.observe(document.body, { childList: true, subtree: true });
}
```

## 6. 错误处理

### 6.1 错误类型

| 错误码 | 说明 | 处理方式 |
|--------|------|----------|
| `COMMENT_BOX_NOT_FOUND` | 未找到评论框 | 提示用户进入项目详情页 |
| `COMMENT_BOX_TIMEOUT` | 评论框加载超时 | 提示用户刷新页面重试 |
| `PUBLISH_FAILED` | 发布失败 | 提示用户手动操作 |
| `ALREADY_PROCESSING` | 操作进行中 | 提示用户等待 |

### 6.2 错误处理代码

```javascript
function handleError(errorCode, message) {
    const errorMessages = {
        'COMMENT_BOX_NOT_FOUND': '未找到评论框，请确保在项目详情页',
        'COMMENT_BOX_TIMEOUT': '评论框加载超时，请刷新页面重试',
        'PUBLISH_FAILED': '发布失败，请手动操作',
        'ALREADY_PROCESSING': '操作进行中，请稍候...'
    };
    
    showToast(errorMessages[errorCode] || message, 'error');
    logger.error(`[AutoComment] ${errorCode}: ${message}`);
}
```

## 10. 注意事项

1. **低耦合原则**：新功能作为独立模块添加，不修改原有核心逻辑
2. **SPA兼容**：需要处理飞书项目的单页应用特性
3. **用户触发**：功能仅在用户点击图标时执行，避免自动运行
4. **富文本处理**：评论框是富文本编辑器，需要正确设置 `contenteditable` 元素的内容
5. **状态管理**：使用 `isProcessing` 状态防止重复点击
