## 2. 核心架构

### 2.1 技术方案

```
远程调试模式 Chrome (端口 9222)
         ↓
Python + Selenium 连接已有 Chrome
         ↓
在 Chrome 中开新标签页处理
         ↓
JavaScript 提取页面信息
         ↓
关闭标签页，保持 Chrome 开启
```

### 2.2 关键设计决策

| 决策 | 原因 |
|------|------|
| 远程调试模式 | 复用已有 Chrome，保留登录状态 |
| chromedriver 放系统目录 | 避免 git 同步大文件 |
| 自动下载 chromedriver | 小白友好，跨电脑使用 |
| 不主动检测登录 | 避免干扰用户登录流程 |

### 2.3 文件结构

```
一键更新表2026-3-5/
├── one_click_update.py          # 主脚本 v1.7
├── 启动Chrome调试模式.bat        # 第一步：启动调试 Chrome
├── 运行脚本.bat                  # 第二步：运行脚本
├── requirements.txt             # Python 依赖
├── 使用说明.md                   # 用户文档
├── 开发与维护文档.md              # 本文档
└── output/                      # 输出目录

系统目录（不在项目中）：
%LOCALAPPDATA%\chromedriver_feishu\chromedriver.exe
%TEMP%\chrome_feishu_debug\
```

## 3. 核心函数说明

### 3.1 chromedriver 自动下载

| 函数 | 说明 |
|------|------|
| `get_chrome_version()` | 获取本地 Chrome 版本（命令行或注册表） |
| `get_chromedriver_download_url()` | 获取对应版本的下载链接 |
| `download_and_extract_chromedriver()` | 下载并解压到系统目录 |
| `check_and_download_chromedriver()` | 主入口：检查版本匹配，不匹配则下载 |

**关键逻辑**：
- Chrome >= 115：使用 Chrome for Testing API
- Chrome < 115：使用 storage.googleapis.com

### 3.2 Chrome 连接

```python
def connect_to_chrome():
    options = Options()
    options.add_experimental_option("debuggerAddress", "127.0.0.1:9222")
    chromedriver_path = check_and_download_chromedriver()
    service = Service(executable_path=chromedriver_path)
    driver = webdriver.Chrome(service=service, options=options)
    return driver
```

**注意**：不要调用 `driver.quit()`，保持 Chrome 开启。

### 3.3 页面信息提取

| 函数 | 说明 |
|------|------|
| `extract_project_info()` | 等待页面加载后提取信息 |
| `extract_with_retry()` | 带重试机制：失败时等待20s，刷新后再等10s |
| `get_process_info_js()` | JavaScript：提取处理信息 |
| `get_aihelp_link_js()` | JavaScript：提取 aihelp 链接 |

### 3.4 链接输入

| 函数 | 说明 |
|------|------|
| `prompt_links()` | 提供三种输入方式 |
| `read_links()` | 解析文本，过滤出有效链接 |

**关键处理**：`raw_text.replace('https://', '\nhttps://')` 解决粘贴换行符丢失问题。

## 5. 修改注意事项

### 5.1 修改提取逻辑

提取逻辑在 JavaScript 字符串中：
- `_GET_PROCESS_INFO_JS`：处理信息提取
- `_GET_AIHELP_LINK_JS`：aihelp 链接提取

修改时注意：
1. JavaScript 在浏览器内执行，使用 `driver.execute_script()`
2. 返回值通过 `return` 语句
3. 测试时在浏览器 Console 中先验证

### 5.2 修改输出字段

输出字段定义在 `OUTPUT_COLUMNS`：
```python
OUTPUT_COLUMNS = [
    "项目名称", "状态", "项目名称2", "空", 
    "日期", "来源", "aihelp链接", "处理信息",
]
```

修改时同步更新：
1. `extract_project_info()` 中的 `result` 字典
2. `使用说明.md` 中的字段说明

### 5.3 修改 chromedriver 下载逻辑

下载逻辑在 `get_chromedriver_download_url()`：
- 新版 Chrome（>= 115）使用 Chrome for Testing API
- 旧版 Chrome 使用 storage.googleapis.com

如果下载失败，检查：
1. API 地址是否变更
2. 网络是否可访问
3. Chrome 版本是否正确获取

## 6. 常见问题排查

| 问题 | 排查步骤 |
|------|----------|
| 连接失败 | 检查 Chrome 是否以调试模式启动（端口 9222） |
| chromedriver 版本不匹配 | 检查 `get_chrome_version()` 返回值，删除旧 chromedriver 重新下载 |
| 提取失败 | 在浏览器 Console 中测试 JavaScript 提取逻辑 |
| 登录卡住 | 不要主动检测登录，让用户在 Chrome 中操作 |
| 粘贴链接识别不全 | 检查 `raw_text.replace('https://', '\nhttps://')` 是否执行 |
